Quản Lý Bộ Nhớ Trong Java

Java sử dụng cơ chế Garbage Collection (GC) để quản lý bộ nhớ tự động. Điều này giúp lập trình viên không phải lo lắng về việc giải phóng bộ nhớ khi không còn sử dụng đối tượng. Bài viết này sẽ giải thích cách Garbage Collection hoạt động trong Java.

Garbage Collection (GC)

Garbage Collection là quá trình tự động thu gom bộ nhớ trong Java. Các đối tượng không còn được tham chiếu sẽ bị GC loại bỏ để giải phóng bộ nhớ. Java sử dụng các thuật toán như Mark and Sweep và Generational Garbage Collection để tối ưu hóa quá trình này.

Ví dụ về Garbage Collection

public class MyClass {
    @Override
    protected void finalize() {
        System.out.println("Object is being garbage collected!");
    }

    public static void main(String[] args) {
        MyClass obj = new MyClass();
        obj = null; // obj không còn tham chiếu
        System.gc(); // Yêu cầu JVM thực hiện GC
    }
}
